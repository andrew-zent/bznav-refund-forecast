# 마케팅 예상결제액 공식 개선 제안

작성일: 2026-04-17
대상: 마케팅팀 / 데이터팀
근거 데이터: Pipedrive (개인+법인), 12개월 완성 코호트 (2024-11 ~ 2025-10)

---

## TL;DR

**현재 공식**: `신청환급금 × 6.5% = 예상결제액`
**문제**: 실측 대비 평균 **+53% 과대추정** (2025-12: 42.8억 forecast vs 실측 예상 ~27~28억)

**제안: 하이브리드 단계별 정밀화 + 추심 풀 기여 분리**

1. **당일 (T+0)**: 신청환급금 × 4.2% — rough estimate
2. **T+7 (신고 확정 후)**: 신고환급액 × 30%로 replace (MAPE 2%)
3. **T+14 (결정 확정 후)**: 결정환급액 × 31%로 확정 (MAPE 1%)
4. **+ 추심 풀 기여**: **최근 3개월 풀 결제 실측 평균** 사용 (현재 ~1.76억/월). 풀 aging 특성상 `잔액×flat율` 방식은 부정확 — 매월 재산정.

**부가 확인**: 신청 후 1년+에 추심으로 추가되는 부분은 apply-cohort tail로는 +0.1%p에 불과. 추심 회수는 **최근 유입(30~60일 이내 결정 건)이 주도**하며, 오래된 풀은 회수율 급락. 따라서 풀 총 잔액은 "규모 지표"로만 보고, 월 회수 forecast는 rolling 평균이 정답.

## 1. 현재 공식의 문제

### 1.1 2025-12 사례

| 항목 | 값 |
|---|---:|
| 신청환급금 (실측) | 668.4억 |
| 예상결제액 (현재 공식: ×6.5%) | **42.8억** |
| 실제 결제 발생 예상 (cohort 만기 기준) | **~28억** |
| **과대 배수** | **약 1.5배 (+53%)** |

### 1.2 근본 원인

현재 6.5% 계수는 "신청환급금 전체가 결제로 연결된다"는 전제로 설계됨. 하지만 실제 파이프라인 흐름은:

```
신청환급금 668억
  ├─ 유효 유입 (정기/추심/법인 active)  ~140억 (21%)
  │    └─ 신청→결제 전환율 ~20%  →  ~28억 결제
  └─ 필터 제외 (실패/취소/테스트/감면only 등)  ~528억 (79%)
       └─ 결제 0에 수렴
```

즉 668억 전체에 일괄 비율을 적용하기보다, **유효 유입의 실측 전환율**을 역산해야 정확.

### 2.3 Tail 분석 — 1년+ 추심 회수 기여도

신청 후 오랜 시간이 지나도 추심 pipeline을 통해 추가 결제가 발생할 수 있음. 12개월 성숙 코호트(2024-11 ~ 2025-10) 실측:

| 기간 | 결제액 | Total 중 비중 | 신청대비 누적 전환율 |
|---|---:|---:|---:|
| M+0 ~ M+5 (6개월) | 174.1억 | 96.7% | 19.65% |
| M+6 ~ M+11 (7~12개월) | 5.2억 | 2.9% | 20.24% (+0.59%p) |
| M+12 ~ M+24 (1년 초과) | 0.8억 | 0.4% | 20.33% (+0.09%p) |

**결론**: M+5 이후 기여도 급감. 1년 기준 tail 추가분은 +0.6%p, 1년+는 +0.1%p 수준으로 사실상 무시 가능. **제안 계수 4.2%는 1년 누적 기준에 정확히 calibrated** (Full 4.26%, 6개월만 보면 4.12%).

### 2.4 Offset별 결제 발생 분포 (신청대비)

```
M+0:  0.08%
M+1:  1.97%
M+2:  9.65%   ← 피크
M+3:  5.60%
M+4:  1.60%
M+5:  0.73%
M+6:  0.26%
M+7~: 각 0.1% 미만
```

→ **결제의 85%가 M+2 ~ M+3에 집중**. 이는 신고→결정→결제 lag 구조에서 나오는 패턴이며, 계절성/프로모션 효과도 이 분포를 대체로 따름.

### 2.5 마케팅 base 기준 계수 환산

- 마케팅이 보는 "신청환급금" 668억 = 필터 전 전체 유입
- 내부 분석 base 140.1억 = 정기+추심+법인 + 실패 제외
- 스케일 비율: **668.4 ÷ 140.1 = 4.77배**
- 마케팅 base 등가 계수:
  - 6개월 기준: 19.65% ÷ 4.77 = 4.12%
  - **1년 기준: 20.24% ÷ 4.77 = 4.24% (≈ 4.2%)**
  - Full 기준: 20.33% ÷ 4.77 = 4.26%

## 3. 제안

### 3.1 Primary 공식 (당일 forecast)

```
예상결제액 = 신청환급금 × 4.2%
```

- 당일 집계 가능 (신청환급금은 day-0에 확보됨)
- MAPE ~5.7%, 월별 편차 ±15% 이내

### 3.2 Secondary 공식 (7일 후 보정)

```
예상결제액 = 신고환급액 × 30%
```

- 신고는 신청 후 3~7일 지연 → T+7 시점부터 사용 가능
- 실측 편차 ±2% 수준 (월별 28~32%, MAPE 2% 미만)
- 2025-12 사례: 신고환급액 71.2억 × 30% = 21.4억 → 실측 누적 21.4억 **정확히 일치**

### 3.3 Tertiary 공식 (사후 정산)

```
매출 확정 = 결정환급액 × 31%
```

- 결정은 신고 후 1~2개월 → 사후 reconciliation 용도
- 12개월 실측 평균 30.82%, 편차 ±1%
- 이게 실질 수수료율 (VAT 포함)

### 3.4 하이브리드 단계별 정밀화 (동일 월 forecast 시간에 따라 정밀해짐)

각 월의 예상결제액은 시간이 지나면서 사용 가능한 정보가 늘어나므로, 단일 공식이 아닌 **단계적 replacement** 구조 권장.

| 시점 | 기준 입력 | 공식 | 정확도 (MAPE) | 비고 |
|---|---|---|---:|---|
| T+0 (당일) | 신청환급금 | × 4.2% | ~5.7% | 신청 즉시 |
| T+7 (1주) | 신고환급액 | × 30% | ~2% | 신고 완료 후 replace |
| T+14 (2주) | 결정환급액 | × 31% | ~1% | 결정 나면 확정 |
| T+60 (2개월) | 결제액 실측 | — | 0% | 실제 집계 |

**운영 로직 예시**

```
def 예상결제액(월):
    if 결정액 존재:  return 결정액 × 31%
    elif 신고액 존재:  return 신고액 × 30%
    else:              return 신청환급금 × 4.2%
```

→ 당일은 rough, 시간 지나면서 자동으로 정밀해지는 구조.

### 3.5 추심 풀 기여 (aging 반영)

추심 pipeline에서 결정은 났지만 아직 결제가 안 된 건들의 합 = **결정금액 풀**. 다만 이 풀은 **uniform하게 회수되지 않음** — aging 동역학 존재.

**Aging 동역학**
- 풀 진입 직후 30~60일: 추심팀 집중 contact → 회수 활발
- 풀 체류 6개월+: 이탈/연락두절/시효 → 회수율 급락
- 즉 월 회수액은 **최근 유입량 × 회수 확률**에 가까움. `잔액 × flat율` 아님.

**현재 풀 상태 (2026-04)**

| 지표 | 값 | 해석 |
|---|---:|---|
| 풀 잔액 (총합) | 151.2억 | **규모 지표**, flat 회수율 적용 비권장 |
| 최근 3개월 실측 풀 결제 평균 | **1.76억/월** | 2026-01(1.90)+02(1.58)+03(1.81)/3 |
| 최근 3개월 잔액 순감소 | -4.5억/월 평균 | 유입 < 회수, 풀 축소 국면 |
| B결정 → 추심 누수율 | 65.5% | 신규 유입 원천 |
| 추심 풀 유입률 | 19.6% | B누수 중 실제 풀 진입 비율 |

**변동성 (24개월 실측)**
- 월 결제액: 최소 0.00 ~ 최대 1.90억, 표준편차 **0.52억**
- 회수율(paid/balance): 0.026% ~ 1.158%, 표준편차 **0.30%p**
- → flat 값 적용하면 편차 ±50% 이상 오차 가능

**권장 forecast 방식**

```
월간 풀 기여 = 최근 3개월 풀 결제 실측 평균 (rolling)
            ≠ 현재 잔액 × flat 회수율
```

- 현재 기준값: **1.76억/월** (2026-01~03 평균)
- 매월 집계 후 재산정 (T+0에 자동 rolling update)
- **선행지표 경보**: B 정기 누수량(신규 유입 원천)이 감소하면 2~3개월 후 풀 결제도 감소 예상

**월간 예상결제액 통합 공식 (수정)**

```
월간 예상결제액 = (신청환급금 × 4.2%) + 풀 최근 3개월 평균 결제
               = 신규 cohort lifetime 기여 + 추심 회수 (rolling 실측)
```

예: 2025-12 → 28.1 + 1.76 = **29.9억** (마케팅 42.8 대비 30% 과대 축소, 실측 정답 28-30억에 근접)

### 3.5 미성숙 코호트 예측 (Offset 기반)

아직 결제가 누적 중인 최근 코호트는 과거 12개월 성숙 코호트의 offset 분포를 적용해 최종 전환율 예측 가능.

**방법**: 신청월 기준 M+n 경과 시점에서,
- 관찰 결제 = 현재까지 실측 누적액
- 예측 남은 결제 = 신청액 × Σ(M+(n+1) ~ M+24 offset rate)
- 예측 최종 = 관찰 + 예측 남은

**현재 In-flight 코호트 예측** (2026-04 기준)

| 신청월 | 신청액 | 경과 | 관찰 결제 | 관찰 % | 예측 남은 | 예측 최종 | 예측 % |
|---|---:|:-:|---:|---:|---:|---:|---:|
| 2025-11 | 98.1 | M+5 | 18.25 | 18.60% | 0.66 | 18.91 | 19.28% |
| 2025-12 | 140.1 | M+4 | 25.44 | 18.16% | 1.98 | 27.42 | **19.57%** |
| 2026-01 | 78.4 | M+3 | 11.90 | 15.18% | 2.36 | 14.26 | 18.19% |
| 2026-02 | 86.2 | M+2 | 7.71 | 8.95% | 7.43 | 15.15 | 17.56% |
| 2026-03 | 95.4 | M+1 | 0.88 | 0.93% | 17.44 | 18.32 | 19.20% |
| 2026-04 | 25.4 | M+0 | 0.04 | 0.15% | 5.14 | 5.18 | 20.40% |

**시사점**
- 2025-12 실측 예상 최종: **27.4억** (← 마케팅 forecast 42.8억 대비 **36% 과대** 재확인)
- 2026-02 같이 이제 막 결제가 시작된 코호트도 17~19% 수렴 예측
- 계절성으로 편차 있긴 하지만 전반적 20% 근방으로 수렴

## 4. 기대 효과

### 4.1 예측 정확도 개선

- 월별 과대추정 +53% → ±6% 로 **9배 개선**
- 2025-12 시뮬레이션: 42.8억 → 28.1억 (실측 ~28억에 근접)

### 4.2 의사결정 영향

- 과대 forecast 기반 마케팅 예산/목표 설정 방지
- 현금흐름 계획 정확도 개선 (월 매출 오차 ±수억원 수준 축소)
- 경영진 리포트 신뢰도 향상

### 4.3 In-flight 코호트 재예측 효과 (2025-12 케이스)

2025-12 기준 세 가지 예측 방식 비교:

| 방식 | 계산 | 결과 |
|---|---|---:|
| 마케팅 현재 (신청환급금 668억 × 6.5%) | — | **42.8억** |
| 제안 공식 (신청환급금 668억 × 4.2%) | — | **28.1억** |
| offset 기반 정밀예측 (내부 cohort 140.1억 기반) | 관찰 25.4 + 예측 남은 2.0 | **27.4억** |
| 실측 누적 (M+4까지) | — | **21.4억 (진행중)** |

**→ 제안 공식(28.1억)과 정밀예측(27.4억) 모두 실측 수렴 방향 일치**. 마케팅 현재(42.8억)만 심하게 과대.

**In-flight cohort의 offset 기반 정밀예측을 별도 대시보드에 추가하면**, 단순 비율 공식 대비 월별 오차 MAPE 2~3% 수준까지 축소 가능.

### B. 왜 세그먼트 가중이 더 정확하지 않은가

세그먼트별 rate 편차보다 월별 rate 편차가 더 크기 때문. 개인 정기 한 세그먼트 내에서도 월마다 17~23% 변동이 있어, 세그먼트 믹스 보정이 오히려 노이즈 증가. 전체 flat rate가 더 robust.

**Aging 반영 — 풀 기여 forecast 방식**
- 풀 총잔액은 **규모 지표**일 뿐, 그 자체에 flat 회수율을 곱해 forecast로 쓰면 안 됨 (aging 무시)
- 실제 월 회수는 **최근 30~60일 유입량 × 회수 확률**에 가까움 (신규 결정이 단기 내 회수되는 구조, 오래된 풀은 회수율 급락)
- Forecast용 풀 기여값 = **최근 3개월 실측 풀 결제 평균 (rolling)**
  - 현재 기준: 2026-01(1.90) + 02(1.58) + 03(1.81) / 3 = **1.76억/월**
- 풀 감소 추세(-4.5억/월) + 회수 가속화 국면이지만, 유입이 줄면 2~3개월 시차 두고 결제도 감소
- **매월 rolling 재산정 필수** (분기 단위로는 편차 흡수 부족)
- **선행지표**: B 정기 누수량 감소 → 2~3개월 뒤 풀 결제 감소 예상
