# 마케팅 예상결제액 공식 개선 제안

작성일: 2026-04-17
대상: 마케팅팀 / 데이터팀
근거 데이터: Pipedrive (개인+법인), 12개월 완성 코호트 (2024-11 ~ 2025-10)

---

## TL;DR

**현재 공식**: `신청환급금 × 6.5% = 예상결제액`
**문제**: 현재 시점 기준 **약 38% 과대추정** (2025-12: 42.8억 forecast vs 실측 예상 ~31억). 2024-11~2025-01 시기에는 6%대가 실제 전환율이었지만, 최근 **4.5% 수준으로 감소 추세** — 원인 진단 필요.

**제안: 하이브리드 단계별 정밀화 + 추심 풀 기여 분리**

1. **당일 (T+0)**: 신청환급금 × 4.7% — rough estimate
2. **T+7 (신고 확정 후)**: 신고환급액 × 30%로 replace (MAPE 2%)
3. **T+14 (결정 확정 후)**: 결정환급액 × 31%로 확정 (MAPE 1%)
4. **+ 추심 풀 기여**: **최근 3개월 풀 결제 실측 평균** 사용 (현재 ~1.76억/월). 풀 aging 특성상 `잔액×flat율` 방식은 부정확 — 매월 재산정.

**부가 확인**: 신청 후 1년+에 추심으로 추가되는 부분은 apply-cohort tail로는 +0.1%p에 불과. 추심 회수는 **최근 유입(30~60일 이내 결정 건)이 주도**하며, 오래된 풀은 회수율 급락. 따라서 풀 총 잔액은 "규모 지표"로만 보고, 월 회수 forecast는 rolling 평균이 정답.

## 1. 현재 공식의 문제

### 1.1 2025-12 사례

| 항목 | 값 |
|---|---:|
| 신청환급금 (실측) | 668.4억 |
| 예상결제액 (현재 공식: ×6.5%) | **42.8억** |
| 실제 결제 발생 예상 (cohort 만기 기준) | **~28억** |
| **과대 배수** | **약 1.5배 (+53%)** |

### 1.2 근본 원인

현재 6.5% 계수는 "신청환급금 전체가 결제로 연결된다"는 전제로 설계됨. 하지만 실제 파이프라인 흐름은:

```
신청환급금 668억
  ├─ 유효 유입 (정기/추심/법인 active)  ~140억 (21%)
  │    └─ 신청→결제 전환율 ~20%  →  ~28억 결제
  └─ 필터 제외 (실패/취소/테스트/감면only 등)  ~528억 (79%)
       └─ 결제 0에 수렴
```

즉 668억 전체에 일괄 비율을 적용하기보다, **유효 유입의 실측 전환율**을 역산해야 정확.

### 2.2 A(지수) Pipeline이 주범

- **전환율 0.01%** — 1886억 유입에 결제는 0.2억뿐, 사실상 **dead deal 공급처**
- **2배 급성장** (94.4억/월 → 193.0억/월) — unfiltered base 팽창의 주요 원인
- **압도적 점유율 54.3%** — 이 pipeline 하나가 전체 분모의 절반
- **만약 A(지수)를 제외하면** unfiltered 전환율: 207.1 / (3,477.6 − 1,886.8) = **13.0%** (현재 5.96%의 2배)

1. **A(지수) pipeline 비중이 49% → 57%로 증가** (2024-11~01 → 2025-08~10)

### 2.11 권장 액션 (마케팅 관점)

- **A(지수) 정체 파악**: 이 pipeline이 어떤 채널/담당자/프로모션에서 유입되는지 확인
  - 전환율 0.01%는 거의 "신청만 받고 실제 진행은 없는" 상태 → 자격 미달 대량 유입
- **B(젠트)-환급 subset 별도 추적**: 전환율 20%의 "진짜 base"로 마케팅 KPI 설정
- **Unfiltered base에 flat 계수 적용 지양**: pipeline 구성비 변동에 따라 계수가 흔들림. 아래 3가지 운영 옵션:
  1. **(권장)** B(젠트)-환급 base × 20% — 가장 안정적 signal
  2. Unfiltered base × 4.7%, 단 월별 재산정 필수
  3. A(지수) + D(취소) 제외 base × 13%

- 월별 오차 평균(MAPE) 5.7%, 최악 +14.8% → **예측 안정성 우수**
- 세그먼트 가중 적용(각 세그 rate 별도) 시 MAPE 7.2%로 오히려 증가 → flat rate가 실용적

### 3.3 Tail 분석 — 1년+ 추심 회수 기여도

신청 후 오랜 시간이 지나도 추심 pipeline을 통해 추가 결제가 발생할 수 있음. 12개월 성숙 코호트(2024-11 ~ 2025-10) 실측:

| 기간 | 결제액 | Total 중 비중 | 신청대비 누적 전환율 |
|---|---:|---:|---:|
| M+0 ~ M+5 (6개월) | 174.1억 | 96.7% | 19.65% |
| M+6 ~ M+11 (7~12개월) | 5.2억 | 2.9% | 20.24% (+0.59%p) |
| M+12 ~ M+24 (1년 초과) | 0.8억 | 0.4% | 20.33% (+0.09%p) |

**결론**: M+5 이후 기여도 급감. 1년 기준 tail 추가분은 +0.6%p, 1년+는 +0.1%p 수준으로 사실상 무시 가능. **제안 계수 4.7%는 1년 누적 기준에 정확히 calibrated** (Full 4.26%, 6개월만 보면 4.12%).

### 3.4 Offset별 결제 발생 분포 (신청대비)

```
M+0:  0.08%
M+1:  1.97%
M+2:  9.65%   ← 피크
M+3:  5.60%
M+4:  1.60%
M+5:  0.73%
M+6:  0.26%
M+7~: 각 0.1% 미만
```

→ **결제의 85%가 M+2 ~ M+3에 집중**. 이는 신고→결정→결제 lag 구조에서 나오는 패턴이며, 계절성/프로모션 효과도 이 분포를 대체로 따름.

### 3.5 마케팅 base 기준 계수 환산

- 마케팅이 보는 "신청환급금" 668억 = 필터 전 전체 유입
- 내부 분석 base 140.1억 = 정기+추심+법인 + 실패 제외
- 스케일 비율: **668.4 ÷ 140.1 = 4.77배**
- 마케팅 base 등가 계수:
  - 6개월 기준: 19.65% ÷ 4.77 = 4.12%
  - **1년 기준: 20.24% ÷ 4.77 = 4.24% (≈ 4.7%)**
  - Full 기준: 20.33% ÷ 4.77 = 4.26%

## 4. 제안

### 4.1 Primary 공식 (당일 forecast)

```
예상결제액 = 신청환급금 × 4.7%
```

- 당일 집계 가능 (신청환급금은 day-0에 확보됨)
- MAPE ~5.7%, 월별 편차 ±15% 이내

### 4.2 Secondary 공식 (7일 후 보정)

```
예상결제액 = 신고환급액 × 30%
```

- 신고는 신청 후 3~7일 지연 → T+7 시점부터 사용 가능
- 실측 편차 ±2% 수준 (월별 28~32%, MAPE 2% 미만)
- 2025-12 사례: 신고환급액 71.2억 × 30% = 21.4억 → 실측 누적 21.4억 **정확히 일치**

### 4.3 Tertiary 공식 (사후 정산)

```
매출 확정 = 결정환급액 × 31%
```

- 결정은 신고 후 1~2개월 → 사후 reconciliation 용도
- 12개월 실측 평균 30.82%, 편차 ±1%
- 이게 실질 수수료율 (VAT 포함)

### 4.4 하이브리드 단계별 정밀화 (동일 월 forecast 시간에 따라 정밀해짐)

각 월의 예상결제액은 시간이 지나면서 사용 가능한 정보가 늘어나므로, 단일 공식이 아닌 **단계적 replacement** 구조 권장.

| 시점 | 기준 입력 | 공식 | 정확도 (MAPE) | 비고 |
|---|---|---|---:|---|
| T+0 (당일) | 신청환급금 | × 4.7% | ~5.7% | 신청 즉시 |
| T+7 (1주) | 신고환급액 | × 30% | ~2% | 신고 완료 후 replace |
| T+14 (2주) | 결정환급액 | × 31% | ~1% | 결정 나면 확정 |
| T+60 (2개월) | 결제액 실측 | — | 0% | 실제 집계 |

**운영 로직 예시**

```
def 예상결제액(월):
    if 결정액 존재:  return 결정액 × 31%
    elif 신고액 존재:  return 신고액 × 30%
    else:              return 신청환급금 × 4.7%
```

→ 당일은 rough, 시간 지나면서 자동으로 정밀해지는 구조.

### 4.5 추심 풀 기여 (aging 반영) ⭐ **핵심 섹션**

추심 pipeline에서 결정은 났지만 아직 결제가 안 된 건들의 합 = **결정금액 풀**. 다만 이 풀은 **uniform하게 회수되지 않음** — aging 동역학 때문에 `잔액 × flat율` 공식은 부정확.

#### 4.5.1 Aging 동역학

- 풀 진입 직후 **30~60일**: 추심팀 집중 contact → 회수 활발 (대부분의 결제가 이 시기에 발생)
- 풀 체류 **6개월+**: 이탈/연락두절/시효 → 회수율 급락
- 즉 월 회수액 ≈ **최근 유입량 × 회수 확률**, 풀 잔액과 직접 비례하지 않음

```
월간 풀 기여 = 최근 3개월 풀 결제 실측 평균 (rolling)
            ≠ 현재 잔액 × flat 회수율
```

- **현재 기준값: 1.76억/월** (2026-01: 1.90 + 02: 1.58 + 03: 1.81 / 3)
- 매월 집계 후 재산정 (T+0에 자동 rolling update)
- 분기 단위 재산정은 편차 흡수 부족 → 월간 필수

#### 4.5.4 선행지표 경보

- 풀 잔액이 유의미하게 감소 (예: 월 -5억 이상): 향후 월 결제 감소 예상
- **B 정기 누수량**(신규 유입 원천)이 줄어들면 **2~3개월 시차 두고** 풀 결제도 감소
- 현재 상태: -4.5억/월 감소 중 + 회수 가속 → **1~2분기 내 월 풀 결제 1억 아래로 떨어질 수 있음**

#### 4.5.5 통합 공식 (수정)

```
월간 예상결제액 = (신청환급금 × 4.7%) + 풀 최근 3개월 평균 결제
               = 신규 cohort lifetime 기여 + 추심 회수 (rolling 실측)
```

예: 2025-12 → 28.1 + 1.76 = **29.9억** (마케팅 42.8 대비 30% 과대 축소, 실측 정답 28~30억에 근접)

### 4.6 미성숙 코호트 예측 (Offset 기반)

아직 결제가 누적 중인 최근 코호트는 과거 12개월 성숙 코호트의 offset 분포를 적용해 최종 전환율 예측 가능.

**방법**: 신청월 기준 M+n 경과 시점에서,
- 관찰 결제 = 현재까지 실측 누적액
- 예측 남은 결제 = 신청액 × Σ(M+(n+1) ~ M+24 offset rate)
- 예측 최종 = 관찰 + 예측 남은

|---|---:|:-:|---:|---:|---:|---:|---:|
| 2025-11 | 98.1 | M+5 | 18.25 | 18.60% | 0.66 | 18.91 | 19.28% |
| 2025-12 | 140.1 | M+4 | 25.44 | 18.16% | 1.98 | 27.42 | **19.57%** |
| 2026-01 | 78.4 | M+3 | 11.90 | 15.18% | 2.36 | 14.26 | 18.19% |
| 2026-02 | 86.2 | M+2 | 7.71 | 8.95% | 7.43 | 15.15 | 17.56% |
| 2026-03 | 95.4 | M+1 | 0.88 | 0.93% | 17.44 | 18.32 | 19.20% |

**시사점**
- 2025-12 실측 예상 최종: **27.4억** (← 마케팅 forecast 42.8억 대비 **36% 과대** 재확인)
- 2026-02 같이 이제 막 결제가 시작된 코호트도 17~19% 수렴 예측
- 계절성으로 편차 있긴 하지만 전반적 20% 근방으로 수렴

### B. 왜 세그먼트 가중이 더 정확하지 않은가

세그먼트별 rate 편차보다 월별 rate 편차가 더 크기 때문. 개인 정기 한 세그먼트 내에서도 월마다 17~23% 변동이 있어, 세그먼트 믹스 보정이 오히려 노이즈 증가. 전체 flat rate가 더 robust.
