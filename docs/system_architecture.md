# 비즈넵 환급 수수료 결제 예측 시스템

## 1. 시스템 개요

비즈넵의 환급 대행 수수료 결제를 자동 예측하는 파이프라인.
Pipedrive CRM 데이터를 기반으로 코호트 분산 모델을 적용하여 향후 5개월 결제액을 예측하고,
GitHub Pages 대시보드로 자동 배포한다.

### 핵심 지표
- **개인 MAPE**: 5.8% (12개월 Walk-Forward 백테스트)
- **예측 범위**: 향후 5개월
- **데이터**: 개인 233,000+건 / 법인 8,100+건
- **자동화**: 매주 월요일 09:00 KST 갱신

---

## 2. 데이터 흐름

```
[Pipedrive API]
    ├─ 개인 (raw-competition.pipedrive.com)
    │     → extract_pipedrive.py → deals_slim.json (75MB, 233K건)
    │
    └─ 법인 (api.pipedrive.com)
          → extract_corp.py → deals_corp_slim.json (2.4MB, 8K건)
                    │

              [model.py]
                ├─ 개인: 4단계 코호트 분산 모델
                ├─ 개인 추심: 채권풀 × 회수율 모델
                └─ 법인: 6개월 이동평균 + 선형추세
                    │
                    ▼
            [forecast.json]
                    │
                    ▼
         [generate_dashboard.py]
                    │
                    ▼
            [index.html → GitHub Pages]
```

## 3. 데이터 추출 (Extract)

### 3.1 개인 Pipedrive (`extract_pipedrive.py`)

**데이터 소스**: raw-competition.pipedrive.com
**API 필드**: 163개 중 9개 커스텀 + 4개 표준 = **13개만 추출**
**최적화**: 원본 4.1GB → slim 75MB (54x 절감)

#### 추출 필드

| 내부 변수명 | Pipedrive 필드 | 용도 |
|------------|---------------|------|
| apply_date | ✔ 신청일자 | 코호트 시작점 |
| filing_date | ✔ 신고일자 | a2f 분산 대상 |
| decision_date | ✍ 결정일자 | f2d 분산 대상 |
| payment_date | 💸 결제일자 | d2p 분산 대상, 최종 결제 시점 |
| apply_amount | ✔ 조회 환급액 | 신청 금액 |
| filing_amount | ✍ 신고 환급액 | 신고 금액 |
| decision_amount | ✍ 결정 환급액 | 결정 금액 |
| payment_amount | 📍 결제금액 | 실제 수수료 결제액 |
| is_only_gam | 감면only 여부 | 필터용 |

#### Incremental Sync

- **첫 실행 (full)**: 전건 ASC 순회 → deals_slim.json 생성
- **이후 (incremental)**: `update_time DESC` 정렬 → `.last_sync.txt` 이전 건 만나면 **즉시 중단**
  - 주간 변경분 수천건만 다운로드 (기존: 23만건 전건 스캔)
- 기존 데이터와 **ID 기준 upsert 병합**

### 3.2 법인 Pipedrive (`extract_corp.py`)

**데이터 소스**: api.pipedrive.com (별도 토큰)
**파이프라인**: 법인-환급(1), 법인-추심(4), 법인-취소(3)

#### 개인과 차이점

| 항목 | 개인 | 법인 |
|------|------|------|
| 결제금액 필드 | 📍 결제금액 (직접 기록) | 없음 → **결정환급액 × 수수료율** 산출 |
| 수수료율 | 해당없음 | 최종수수료율 필드 (94.5% 30%, 나머지 20~28%) |
| 결정일자 | 있음 → 코호트 모델 가능 | 없음 → 간이 모델 사용 |
| 규모 | ~15억/월 | ~0.25억/월 (개인의 ~1.5%) |

#### 결제금액 산출 로직

```
헥토계좌 + 헥토카드 > 0 이면:
    payment_amount = 헥토 합계
아니면:
    payment_amount = decision_amount × (fee_rate / 100)
    fee_rate 없으면 기본 30% 적용
```

## 5. 예측 모델 상세

### 5.1 개인 정기: 4단계 코호트 분산 모델

환급 프로세스가 **신청 → 신고 → 결정 → 결제** 4단계를 거치며,
각 단계 간 금액 전환율의 래그(월) 분포를 학습하여 미래 결제를 예측한다.

#### 수학적 정의

**월 인덱스**: `ym = year × 12 + (month - 1)`

**코호트 분산 비율**:

```
P(stage, offset) = mean_over_cohorts(
    target_amount[cohort_month + offset] / source_amount[cohort_month]
) × 100
```

- cohort_month: 소스 단계의 발생 월
- offset: 소스→타겟 단계까지 경과 개월 수 (0, 1, 2, ...)
- Rolling 6개월 윈도우 내 코호트만 사용
- 최소 금액 1억 미만 코호트 제외

#### 3개 분산 체인

**a2f (신청→신고)**, max_offset=4:

| offset | 비율 | 의미 |
|--------|------|------|
| 0개월 | 29.02% | 같은 달 신고 |
| 1개월 | 29.31% | 다음 달 신고 |
| 2개월 | 1.76% | |
| 3개월 | 0.75% | |
| 4개월 | 0.26% | |

**f2d (신고→결정)**, max_offset=4:

| offset | 비율 | 의미 |
|--------|------|------|
| 0개월 | 14.22% | 같은 달 결정 |
| 1개월 | 41.79% | 가장 큰 비중 |
| 2개월 | 37.98% | |
| 3개월 | 3.16% | |
| 4개월 | 2.08% | |

**d2p (결정→결제)**, max_offset=3:

| offset | 비율 | 의미 |
|--------|------|------|
| 0개월 | 18.61% | 같은 달 결제 |
| 1개월 | 14.29% | |
| 2개월 | 1.54% | |
| 3개월 | 0.07% | |
| **합계** | **34.51%** | 나머지 65.49%는 미결제(누수) |

#### 예측 산식

미래 월 `T`의 정기 결제 예측:

```
pay_regular(T) = Σ dec(T - offset) × d2p[offset] / 100
                 offset ∈ {0, 1, 2, 3}
```

여기서 `dec(m)`이 미래 월이면 체인으로 추정:

```
dec(m) = Σ fil(m - offset) × f2d[offset] / 100
fil(m) = Σ app(m - offset) × a2f[offset] / 100
app(m) = 최근 3개월 평균 (실측 없을 때)
```

#### 시즌 보정

```
adjusted(T) = total(T) × (1 + season_adj[month_of(T)])
```

| 월 | 보정 | 근거 |
|----|------|------|
| 1월 | 0% | 연초 |
| 2월 | -5% | 설 연휴 영업일 감소 |
| 3월 | +10% | 1Q 폭증 (raw 오차 -1%이므로 +20%에서 축소) |
| 4~5월 | 0% | 평월/종소세 신고기 |
| 6~7월 | **0%** | 종소세 결정지연 → **결제 이월**, d2p 래그가 이미 반영하므로 **-10% 제거** |
| 8~11월 | 0% | 평월 |
| 12월 | +5% | 연말 결제집중 (기존 +15%에서 축소) |

> **핵심 인사이트**: 종소세 시즌(6~7월) 결정 지연은 결제가 줄어드는 게 아니라 밀리는 것.
> d2p 래그 분포가 이미 이를 포착하고 있어 -10%를 또 씌우면 이중 차감.
> 시즌 보정 최적화로 MAPE 6.27% → 4.34% 개선 (12개월 보정 후 기준).

### 5.2 개인 추심: 채권풀 기반 모델

단순 이동평균(MA3) 대신, **잔여 채권풀 × 월간 회수율** 기반으로 예측.
B(정기) 파이프라인의 결제 누수가 추심 풀의 유입으로 연결되는 구조를 반영.

#### 구조

```
B 결정 → 34.5% 결제 (정기 d2p)
       └→ 65.5% 누수 → 18.7%가 추심풀 유입
                          │
                   추심풀 149.4억
                          │
                   × 1.109%/월 = 추심 결제
```

#### 핵심 변수

| 변수 | 값 | 산출 방식 |
|------|-----|---------|
| 풀 잔액 (col_pool) | 149.4억 | 미결제 추심 건 결정액 합계 |
| 월간 회수율 (col_util_rate) | 1.109% | 최근 3개월: 결제액 / 풀잔액 평균 |
| B 누수율 (col_leak_pct) | 65.49% | 100% - d2p 합계(34.51%) |
| 누수→풀 유입률 (col_inflow_rate) | 18.7% | 최근 6개월: 풀유입 / B누수 평균 |

#### 예측 산식

매월 시뮬레이션으로 풀을 갱신하며 결제액 산출:

```
for each future month i:
    b_dec = 해당 월 B 결정 예측값
    inflow = b_dec × 65.5% × 18.7%        # B 누수 → 풀 유입
    payment = pool × 1.109%                 # 풀에서 회수
    pool = max(pool + inflow - payment, 0)  # 풀 갱신

pay_collection(T) = pool(T) × util_rate
```

#### 채권풀 에이징 분포 (현재)

| 에이징 | 잔액 | 비중 |
|--------|------|------|
| ~3개월 | 0.1억 | 0.1% |
| 3~6개월 | 14.0억 | 9.4% |
| 6~12개월 | 26.0억 | 17.4% |
| 12~24개월 | 53.7억 | 35.9% |
| 24개월+ | 55.7억 | 37.3% |

> **도메인 컨텍스트**: 2026-01부터 추심 인력 투자 증가 → 6개월 이전 오래된 건까지 적극 회수.
> 2025년 월평균 0.7억 → 2026년 1.76억으로 구조적 점프.
> 풀이 2026-02부터 순감소 (회수 > 유입).

### 5.3 법인: 간이 모델

법인은 데이터가 적고(8,174건) 월 결제 변동이 커서(MAPE ~50%) 코호트 모델 대신
**6개월 이동평균 + 선형추세**를 사용.

#### 산식

```
최근 6개월 데이터: [v₁, v₂, ..., v₆]
선형회귀: y = slope × x + intercept

예측(months_ahead):
    pred = intercept + slope × (5 + months_ahead)
    return max(pred, 0)
```

#### 현재 값

| 항목 | 정기 | 추심 |
|------|------|------|
| 6개월 평균 | 0.243억 | 0.014억 |
| 월간 추세 | -0.015억 | +0.010억 |

## 6. 백테스트 검증

### 방법: 12개월 Walk-Forward

각 과거 월 T에 대해:
1. T-1 시점까지의 데이터로 분산 재학습 (refit)
2. T월 결제 예측
3. 실제값과 비교

```
MAPE = mean(|예측 - 실제| / 실제 × 100) over 12 months
```
